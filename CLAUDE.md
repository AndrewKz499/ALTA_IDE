# CLAUDE.md

Инструкции для ассистента (Claude) в этом репозитории.

## Цель проекта

Дизайн-система и кликабельные прототипы по цепочке
**Figma → токены → GitHub → компоненты → экраны → сценарии**.
Задача — превращать дизайн из Figma в переиспользуемые компоненты и собирать из
них интерактивные прототипы, которые можно открыть в браузере.

## Порядок работы

Двигайся строго по цепочке, снизу вверх:

1. **Токены** (`tokens/`) — сначала заводим/обновляем дизайн-токены.
2. **Компоненты** (`components/`) — собираем компоненты, используя только токены.
3. **Экраны** (`prototypes/`) — компонуем экраны из готовых компонентов.
4. **Сценарии / flow** (`prototypes/`) — связываем экраны в кликабельные сценарии.

Не забегай вперёд: нельзя верстать экран из компонента, которого ещё нет, или
хардкодить значение вместо токена.

## Соглашения

- **Один компонент — одна папка** внутри `components/`, с собственным `README.md`,
  описывающим назначение, API/пропсы и пример использования.
- **Семантические токены только через CSS-переменные** (`var(--token-name)`).
  Компоненты ссылаются на семантические токены, а не на primitive напрямую.
- **Никаких хардкод-значений**: цвета, размеры, отступы, радиусы, тени — только
  из токенов. Магических чисел и литеральных `#hex` в компонентах быть не должно.
- Разметка и стили — статические HTML/CSS/JS, совместимые с GitHub Pages.

## Где что лежит

- `tokens/` — дизайн-токены (источник истины для стилей).
- `components/` — компоненты, по одному на папку, с README.
- `prototypes/` — экраны и кликабельные сценарии.
- `references/` — референсы и выгрузки из Figma.
- `assets/` — иконки, изображения, шрифты.
- `docs/` — документация, включая `docs/glossary.md`.

## Стиль коммитов

[Conventional Commits](https://www.conventionalcommits.org/), кратко. Заголовок —
на английском в нижнем регистре:

- `feat: add button component`
- `fix: correct spacing token in card`
- `docs: update glossary`
- `chore: setup gitignore`

Типы: `feat`, `fix`, `docs`, `style`, `refactor`, `chore`.

## Обязательные правила (проверяемые)
1. Экраны собираются только из существующих компонентов `components/`.
2. Не создавать локальные копии CSS компонентов внутри `prototypes/`.
3. Не менять разметку/стили переиспользуемого компонента внутри экрана.
4. Если нужного компонента нет — остановить сборку экрана и зафиксировать
   пробел (gap analysis в `docs/gap/`), сначала добавить компонент в `components/`.
5. Перед созданием любого компонента проверить `docs/component-registry.md`.
6. Все цвета, типографика, отступы, границы, радиусы, тени — только из
   `tokens/tokens.css`. Сырые HEX/RGB/px при наличии токена запрещены.
7. Папки компонентов зеркалят иерархию имён Figma.
8. Каждый компонент содержит: template.html, CSS, JS (если нужно поведение),
   demo.html, README.md. Реализовать только релевантные состояния.
9. Все страницы работают через `file://` и на GitHub Pages.
10. Только относительные URL. Не использовать пути, начинающиеся с `/`.
11. Не использовать `fetch()` для подгрузки локальных HTML-фрагментов.
12. Сохранять клавиатурную навигацию и видимый focus (`:focus-visible`).
13. Использовать термины из `docs/glossary.md` в именах, комментариях, отчётах.
14. Не добавлять npm-пакеты и фреймворки без явного требования задачи.

## Definition of done
Задача завершена только когда:
- реализованы все запрошенные состояния;
- demo/экран работает, нет ошибок в консоли;
- нет дублирования стилей компонентов в `prototypes/`;
- README и реестры обновлены;
- относительные пути работают через `file://`.
