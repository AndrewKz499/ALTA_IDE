# Глоссарий

Единый словарь терминов проекта и соответствие названий между Figma и кодом.

## Термины

- **Дизайн-токен** — именованная переменная, хранящая одно значение дизайна
  (цвет, размер, отступ, радиус, тень и т. п.). Единый источник истины для
  стилей, переиспользуемый в токенах, компонентах и экранах.

- **Primitive-токен** — «сырое» базовое значение без привязки к смыслу
  (например, `color.blue.500`, `space.4`). Не используется в компонентах напрямую.

- **Semantic-токен** — токен со смысловым назначением, ссылающийся на primitive
  (например, `color.action.primary`, `space.inset.md`). Именно его подключают
  компоненты через CSS-переменные.

- **Компонент** — переиспользуемый элемент интерфейса (кнопка, поле, карточка),
  собранный на токенах. В репозитории живёт в собственной папке с README.

- **Component set** — набор вариантов одного компонента (состояния, размеры,
  типы) как единая сущность; в Figma — variants, в коде — пропсы/модификаторы.

- **Экран** — цельная страница интерфейса, скомпонованная из компонентов.

- **Сценарий / flow** — последовательность связанных экранов, показывающая путь
  пользователя (переходы по кликам).

- **Прототип** — кликабельная сборка экранов и сценариев, которую можно открыть
  в браузере (публикуется через GitHub Pages).
